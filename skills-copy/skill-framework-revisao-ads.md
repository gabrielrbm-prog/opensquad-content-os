# Framework de Revisão de ADS — Fluxo Completo (Macro + Micro)

> Último passo antes de entregar os criativos. Aplicado DEPOIS de montar todos os ads (hooks, body, CTA).
> Adaptado do Framework de Revisão v1.1 do Agente 10 (VSL) para contexto de anúncios curtos (30s-2min vídeo + estáticos + carrosséis).

---

## COMO USAR ESTE DOCUMENTO

A revisão de criativos de ADS tem duas fases sequenciais:

- **FASE 1 — REVISÃO MACRO:** Verifica se a ESTRUTURA de cada criativo funciona (falhas críticas, congruência, momentum)
- **FASE 2 — REVISÃO MICRO:** Verifica se a ESCRITA funciona (verbos, especificidade, ritmo, anti-leiturabilidade, clichês)

A ordem é MACRO primeiro, MICRO depois. Não faz sentido polir frases de um ad que vai ser reestruturado.

**Diferença chave vs. revisão de VSL:** Ads são peças CURTAS (30s-2min). Cada problema de revisão tem impacto proporcionalmente MAIOR — uma frase fraca em um ad de 30s é catastrófica, enquanto na VSL de 30min é apenas uma zona fria corrigível.

---

## FASE 1: REVISÃO MACRO

Objetivo: Garantir que cada criativo funciona como um TODO — hook, body e CTA fluem como peça única.

---

### PASSO MACRO 01 — CUBA PASS (Diagnóstico de Falhas Críticas)

Objetivo: Identificar 4 tipos de problemas que MATAM conversão. Passar por cada criativo e marcar problemas encontrados.

#### C — CONFUSING (Confuso)

**O que procurar em ads:**

- Mecanismo explicado de forma confusa (em ad curto, precisa ser ULTRA simples)
- Termos técnicos sem explicação — "Fair Value Gap" sem contexto
- Frase longa demais para ad (15+ palavras em peça de 30s = confuso)
- Salto lógico entre hook e body (hook fala de uma coisa, body de outra)

**Checklist:**
- [ ] Cada frase é compreensível em UMA leitura/escuta?
- [ ] Termos técnicos têm explicação natural (não parênteses acadêmicos)?
- [ ] Hook → Body → CTA tem lógica sequencial clara?
- [ ] Alguém de fora do nicho entende a ideia geral?

#### U — UNBELIEVABLE (Inacreditável)

**O que procurar em ads:**

- Promessa sem prova: "Você vai ganhar R$5k" sem dado ou depoimento
- Números redondos que parecem inventados: usar números QUEBRADOS (527% > 500%)
- Alegações ambiciosas sem ancoragem em mecanismo
- Avatar fazendo alegações que não combinam com quem ele é (UGC citando estudo de Harvard)

**Checklist:**
- [ ] Cada promessa tem pelo menos 1 prova (número, depoimento, demo)?
- [ ] Números são quebrados (específicos), não redondos?
- [ ] As alegações são críveis NA BOCA de quem fala (congruência avatar)?
- [ ] Há pelo menos 1 elemento de prova no body?

#### B — BORING (Chato)

**O que procurar em ads:**

- Blocos técnicos sem tradução emocional (especialmente em ad curto — CADA segundo conta)
- Informação que não ajuda a decisão de clique (cortar sem piedade)
- Ritmo monótono — todas as frases com mesmo tamanho e tom
- Falta de Disparo de Dopamina em Long Pack (perde atenção)

**Checklist:**
- [ ] Cada bloco tem impacto emocional ou é pura informação?
- [ ] Se eu cortar esse bloco/frase, perco conversão? (Se não → corta)
- [ ] Long Pack tem pelo menos 1 Disparo de Dopamina?
- [ ] O ritmo varia (frases curtas + longas)?

#### A — AWKWARD (Estranho)

**O que procurar em ads:**

- Transição abrupta entre hook e body (parece colagem)
- Frase de Aterrissagem que não abre loop (está ali mas não prende)
- CTA que surge do nada (sem ponte do body para o pedido)
- Tom que muda bruscamente (hook provocador → body corporativo)

**Checklist:**
- [ ] Hook → Frase de Aterrissagem → Body flui naturalmente?
- [ ] Body → CTA tem transição orgânica?
- [ ] O tom é consistente do início ao fim?
- [ ] Lendo/ouvindo em voz alta, nada trava?

**ENTREGA DO PASSO MACRO 01:**
```
CUBA PASS — [Nome do Criativo]

CONFUSING: [X] problemas → [X] corrigidos
UNBELIEVABLE: [X] problemas → [X] corrigidos
BORING: [X] problemas → [X] corrigidos
AWKWARD: [X] problemas → [X] corrigidos
```

### PASSO MACRO 02 — TESTE DE CONGRUÊNCIA

> Este passo é CRÍTICO para ads e NÃO existe da mesma forma na revisão de VSL. Em ads, a congruência entre avatar, linguagem e dados é o fator #1 de credibilidade.

Objetivo: Garantir que TUDO no ad é crível vindo de quem fala.

**3 dimensões de congruência:**

#### 2A — Congruência Avatar × Linguagem

O avatar fala como falaria na vida real?

| Avatar | Linguagem CONGRUENTE | Linguagem INCONGRUENTE |
|--------|---------------------|----------------------|
| UGC mulher 40+ | "Menina, eu não acreditei" | "Segundo estudos recentes da Nature..." |
| Médico/Expert | "Na minha experiência clínica de 15 anos..." | "Mano, tu não vai acreditar" |
| Jovem creator | "Gente, olha isso" | "Diante dos dados apresentados..." |
| Narração sem rosto | Tom neutro, informativo | Gírias pessoais, primeira pessoa emocional |

**Checklist:**
- [ ] Se eu fechar os olhos e ouvir, parece alguém REAL falando?
- [ ] O vocabulário combina com a idade/perfil do avatar?
- [ ] O nível de formalidade é consistente?
- [ ] Expressões usadas são naturais para esse tipo de pessoa?

#### 2B — Congruência Avatar × Dados

O avatar SABERIA essas informações?

| Avatar | PODE citar | NÃO PODE citar |
|--------|-----------|----------------|
| UGC comum | Experiência pessoal, resultado próprio | Estatísticas médicas, estudos científicos |
| Médico | Dados clínicos, estudos, mecanismos | Gírias de internet, memes |
| Expert do nicho | Dados da própria operação, resultados de alunos | Dados que não são dele/publicamente verificáveis |
| Narração | Qualquer dado (é voz institucional) | Experiência pessoal em primeira pessoa |

**Checklist:**
- [ ] Cada dado/estatística é crível vindo de quem fala?
- [ ] Se o avatar cita um estudo, ele teria acesso a esse estudo?
- [ ] Números de resultado são do PRÓPRIO avatar ou claramente referenciados?

#### 2C — Congruência Ad × Destino

O que o ad promete é o que o destino entrega?

**Checklist:**
- [ ] A promessa do hook é compatível com o que a VSL/página cobre?
- [ ] O tom do ad é compatível com o tom do destino?
- [ ] O nível de expectativa criado pelo ad pode ser cumprido?
- [ ] Se alguém clicar esperando X, vai encontrar X (ou algo melhor)?

**ENTREGA DO PASSO MACRO 02:**
```
CONGRUÊNCIA — [Nome do Criativo]

Avatar × Linguagem: [OK / X problemas → X corrigidos]
Avatar × Dados: [OK / X problemas → X corrigidos]
Ad × Destino: [OK / X problemas → X corrigidos]
```

### PASSO MACRO 03 — MOMENTUM EMOCIONAL (Long Pack Only)

> Este passo se aplica APENAS a Long Packs (1-2min). Short Packs (30-45s) são curtos demais para ter zonas frias significativas.

Objetivo: Garantir que o ad mantém temperatura emocional. Detectar e corrigir zonas onde o prospect esfria.

**Classificar cada bloco com intensidade emocional:**

- **ALTA** (vermelho) = Dor visceral, desejo intenso, revelação impactante, Disparo de Dopamina
- **MÉDIA** (amarelo) = Explicação com benefício, educação engajada, prova com contexto emocional
- **BAIXA** (cinza) = Informação técnica pura, mecânica sem benefício, dados sem contexto

**PROBLEMA CRÍTICO: 2+ blocos cinza seguidos (em ad, o limiar é MENOR que em VSL)**

**Corrigir zonas frias:**
- **Estratégia A:** Intercalar com Disparo de Dopamina (bloco 26)
- **Estratégia B:** Comprimir bloco técnico — em ad curto, 1 frase é suficiente para transmitir o princípio
- **Estratégia C:** Adicionar mini-agitação entre blocos: "E você continua sem saber disso."

**Padrão ideal para Long Pack:**
```
Hook — ALTA
Frase de Aterrissagem — ALTA
Bloco 1 (problema/identificação) — ALTA
Bloco 2 (mecanismo/solução) — MÉDIA
Disparo de Dopamina — ALTA (pattern interrupt)
Bloco 3 (prova/demo) — MÉDIA
Bloco 4 (benefício/future pacing) — ALTA
CTA — ALTA
Frase Final (Autoridade Humanizada ou Push-and-Pull) — ALTA
```

**Regras:**
- Nunca mais de 1 bloco cinza seguido (em ad, 2 cinzas seguidas = morte)
- O ad deve COMEÇAR em alta e TERMINAR em alta
- Disparos de Dopamina são "injeções de alta" no meio

**ENTREGA DO PASSO MACRO 03:**
```
MOMENTUM — [Nome do Criativo]

MAPA: [Hook:ALTA] → [Aterrissagem:ALTA] → [B1:cor] → [B2:cor] → ...
Zonas frias: [X] → Corrigidas: [X]
```

## FASE 2: REVISÃO MICRO

Objetivo: Garantir que a ESCRITA funciona — cada frase é clara, impactante e fluida. Aplicada DEPOIS da revisão macro.

### FILTRO DE QUALIDADE — GOVERNA TODA DECISÃO DE EDIÇÃO

Antes de aceitar ou rejeitar QUALQUER mudança, passar pelo triplo filtro:

- **Impacto:** Essa mudança aumenta o impacto emocional?
- **Clareza:** O ouvinte/leitor entende imediatamente?
- **Fluidez:** Flui naturalmente ao ser lido/ouvido em voz alta?

**Cortar palavras É UMA FERRAMENTA — não um objetivo.** Só corte se aumenta impacto, mantém clareza e melhora fluidez.

---

### PASSO MICRO 01 — VERIFICAÇÃO DE VERBOS

Verbos mortos (fazer, ter, ser, estar, ficar, pegar, dar, colocar, ver, falar) foram substituídos por verbos com carga visual/emocional?

- Ação física: "pegar" → agarrar/sacar/embolsar
- Estado emocional: "ficar ansioso" → sufocado pela ansiedade
- Transformação: "fazer mudança" → virar o jogo

**Regra:** NÃO sacrificar clareza por verbo bonito. Se "fazer" é mais claro → mantém. Em ads especialmente, clareza > beleza.

### PASSO MICRO 02 — VERIFICAÇÃO DE ESPECIFICIDADE

Adjetivos mortos (muito, grande, alto, incrível, ótimo, fantástico) foram substituídos por especificidade concreta?

**5 substituições:** número exato, comparação concreta, descrição sensorial, contexto emocional, metáfora visceral.

**Regra:** Genérico = invisível. Específico = sentido. Em ads, cada palavra genérica é um segundo desperdiçado.

### PASSO MICRO 03 — VERIFICAÇÃO DE VOZ ATIVA E GORDURA

1. Frases em voz passiva que seriam mais fortes em ativa
2. Palavras-muleta: basicamente, literalmente, tipo, meio, "que" desnecessário
3. Em ads, CADA palavra conta. Se não agrega → corta.

**Regra especial para ads:** A tolerância à gordura é ZERO. Em VSL de 30min, uma palavra a mais passa despercebida. Em ad de 30s, uma palavra a mais = 3% do tempo total desperdiçado.

### PASSO MICRO 04 — VERIFICAÇÃO DE RITMO

- Ler em voz alta (verificação mais importante)
- Variação de comprimento de frase (curta-longa-curta)
- Pausas naturais antes de frases de impacto
- Disparos de Dopamina quebram o ritmo propositalmente (isso é BOM)

**Regra de ouro:** Copy não é lida — é OUVIDA mentalmente. Se travou na leitura em voz alta → reescrever.

### PASSO MICRO 05 — ANTI-LEITURABILIDADE

Detectar e corrigir sinais de que a IA escreveu o ad em vez de uma pessoa real:

| Sinal de Leiturabilidade | Correção para Ad |
|---|---|
| Frases com 15+ palavras | Quebrar em 2 frases curtas (em ad, tolerância é MENOR que VSL) |
| Conectivos formais ("entretanto", "ademais") | Substituir por "mas", "e", "só que" ou CORTAR (conectores enfraquecem ads) |
| Tom de "apresentação corporativa" | Reescrever como fala natural do avatar |
| Simetrias perfeitas / triplicações polidas | Quebrar a simetria com reformulação natural |
| Vocabulário que o prospect não usa | Substituir por palavras do cotidiano dele |

**Teste rápido:** Se parece um texto de blog corporativo, reescrever.

**Regra especial para Disparos de Dopamina:** Verificar que NENHUM disparo tem conector. Se tem → remover o conector ou reescrever a frase inteira.

### PASSO MICRO 06 — VERIFICAÇÃO ANTI-CLICHÊ

Existem frases que o cérebro processaria no piloto automático?

**Teste do contexto:**
1. O público já ouviu essa frase nesse contexto? Se sim → provavelmente clichê
2. O pré-contexto constrói essa frase como conclusão natural? Frase solta = clichê
3. Se tirar o contexto, a frase funciona igual? → genérica, provavelmente clichê

**Em ads, clichês são LETAIS** porque o tempo é curto e cada frase precisa ter impacto máximo. A tolerância é menor que em VSL.

### PASSO FINAL — TESTE DE COMPARTILHAMENTO

> **"Alguém enviaria isso pra um amigo?"**

Este é o teste que separa ads que vendem de ads que ESCALAM. Aplicado na peça COMPLETA (hook + body + CTA), não em partes isoladas.

**Lógica:** Ads que escalam são ads que o público compartilha organicamente. Se o criativo não é bom o suficiente pra alguém mandar pro WhatsApp de um amigo dizendo "olha isso", ele pode converter — mas dificilmente vai escalar.

**O que faz alguém compartilhar um ad:**
- Hook tão surpreendente que a pessoa quer mostrar pra alguém
- Prova tão impressionante que gera "não acredito"
- Identificação tão forte que a pessoa pensa "isso é sobre mim/meu amigo"
- Fechamento tão emocional que toca genuinamente

**Como aplicar:** Para cada criativo finalizado, perguntar: "Se eu visse isso no feed, mandaria pra alguém?" Se a resposta é NÃO → o ad pode funcionar mas não vai escalar. Considerar fortalecer o elemento mais fraco (hook, prova, identificação ou fechamento).

## REGRAS ESPECÍFICAS PARA ESTÁTICOS E CARROSSÉIS

A revisão macro/micro se aplica também a estáticos e carrosséis, com adaptações:

### Estáticos (imagem única)
- CUBA Pass: focar em C (headline confusa?) e U (promessa crível?)
- Congruência: formato visual × avatar × headline — tudo bate?
- Micro: cada palavra da headline é insubstituível? Se cortar alguma, perde força?
- Anti-clichê: headlines clichê são o erro #1 em estáticos

### Carrosséis
- CUBA Pass: aplicar em cada slide individualmente + no fluxo
- Momentum: cada slide deve manter ou aumentar tensão (NUNCA cair)
- Transições entre slides: a última linha de um slide puxa para o próximo?
- Último slide = CTA forte (não pode ser slide fraco)
