# Skill: Estrutura Invisível — Análise Psicológica de Criativos

> Framework para desconstruir anúncios de concorrentes e conteúdos orgânicos virais, mapeando a **função psicológica** de cada linha/elemento. O objetivo é extrair a estrutura que faz o criativo funcionar — não o texto em si, mas a engenharia por trás dele.

> **Inspiração:** Método da Amanda Caiate ("Estrutura Invisível") — analisar o que cada linha FAZ, não o que ela DIZ. Deletar o original, ficar com o mapa. Reescrever a partir das anotações.

---

## QUANDO USAR

Esta skill é acionada no **Passo 0 — Benchmark de Criativos** (etapa 3.3 do Protocolo), aplicada sobre os melhores criativos encontrados na pesquisa orgânica e na análise de validados.

Também pode ser usada sob demanda quando o orquestrador fornecer um criativo específico para análise.

---

## PROCESSO EM 5 ETAPAS

### ETAPA 1 — SELECIONAR PEÇAS PARA ANÁLISE

```
CRITÉRIOS DE SELEÇÃO:
→ Pegar os TOP 3-5 criativos entre:
  - Orgânicos virais (alto engajamento: views, likes, comentários)
  - Ads validados (rodando há semanas/meses na biblioteca Meta)
  - Ads de nichos vizinhos que chamaram atenção
  - Ads do mesmo nicho nos EUA (modelagem cross-country —
    americanos estão 6-12 meses à frente, adaptar US→BR
    é altamente eficaz para encontrar formatos/estruturas
    que ainda não existem no mercado brasileiro)

INPUTS ACEITOS:
→ Texto de ad (copy de estático ou roteiro de vídeo)
→ Transcrição de vídeo ad
→ Descrição detalhada de um ad (quando o texto não está disponível)
→ Print/screenshot com texto visível

REGRA: Mínimo 3 peças, máximo 5 por rodada de benchmark.
       Qualidade da análise degrada com volume excessivo.
```

---

### ETAPA 2 — DESCONSTRUÇÃO LINHA A LINHA

Para cada criativo selecionado, analisar **cada linha/elemento** e anotar:

```
TEMPLATE DE ANOTAÇÃO POR LINHA:

[LINHA ORIGINAL]: "texto exato da linha"
├── FUNÇÃO PSICOLÓGICA: O que essa linha FAZ no cérebro do leitor?
│   (ex: gera identificação, abre loop, prova social, pattern interrupt,
│    agita dor, cria urgência, reforça autoridade, elimina objeção)
├── PEÇA LEGO EQUIVALENTE: Qual bloco do Sistema Lego corresponde?
│   (ex: Situação Reconhecível, Mecanismo Único, Depoimento/UGC, etc.)
│   Se NÃO corresponde a nenhuma peça existente → marcar como "NOVA"
├── TÉCNICA IDENTIFICADA: Qual técnica de copy está sendo usada?
│   (ex: loop aberto, contraste, especificidade numérica, piggybacking,
│    disparo de dopamina, nome chiclete, super estrutura)
└── FORÇA (1-5): Quão bem essa linha executa sua função?
    1 = fraca/genérica, 5 = magistral/impossível de ignorar
```

**Exemplo de desconstrução:**

```
[LINHA 1]: "Eu tinha vergonha de sair de casa"
├── FUNÇÃO: Identificação + agitação de dor emocional
├── PEÇA LEGO: Situação Reconhecível (Grupo ABERTURA)
├── TÉCNICA: Vulnerabilidade do avatar + especificidade emocional
└── FORÇA: 4/5

[LINHA 2]: "Até que minha vizinha me falou de um truque que ela viu num programa americano"
├── FUNÇÃO: Abre loop (que truque?) + Super Estrutura ("programa americano") + Prova Social implícita (vizinha)
├── PEÇA LEGO: Discovery Moment (Grupo SOLUÇÃO)
├── TÉCNICA: Loop aberto + piggybacking + Super Estrutura
└── FORÇA: 5/5

[LINHA 3]: "267 mil pessoas já testaram nos últimos 3 meses"
├── FUNÇÃO: Pattern interrupt + prova social massiva
├── PEÇA LEGO: Números Específicos (Grupo PROVA) — mas DESCONECTADA do contexto
├── TÉCNICA: Disparo de Dopamina (frase jogada, sem conector)
└── FORÇA: 5/5 — número quebrado + sem transição = altamente crível + quebra padrão
```

---

### ETAPA 3 — MAPA DA ESTRUTURA

Depois de desconstruir todas as linhas, montar o **mapa resumido** da estrutura:

```
TEMPLATE DE MAPA:

═══════════════════════════════════════════════
MAPA DE ESTRUTURA INVISÍVEL
Criativo: [identificação/nome]
Fonte: [orgânico/ad validado/nicho vizinho]
Plataforma: [onde foi encontrado]
Performance estimada: [views, tempo rodando, engajamento]
═══════════════════════════════════════════════

SEQUÊNCIA DE FUNÇÕES:
1. [Função psicológica da linha 1] → Peça: [X]
2. [Função psicológica da linha 2] → Peça: [X]
3. [Função psicológica da linha 3] → Peça: [X]
...

ESTRUTURA LEGO RESUMIDA:
→ [Peça 1] → [Peça 2] → [Peça 3] → ... → [CTA]
  (ex: Situação Reconhecível → Discovery Moment → Disparo de Dopamina
   → Mecanismo Único → Números Específicos → CTA Direto)

RECEITA MAIS PRÓXIMA:
→ [Qual das 6 receitas validadas mais se aproxima?]
→ Diferenças: [o que esse criativo faz diferente da receita padrão]

TÉCNICAS DESTACADAS:
→ [Técnica 1]: como foi aplicada e por que funciona
→ [Técnica 2]: como foi aplicada e por que funciona

PEÇAS NOVAS IDENTIFICADAS:
→ [Se alguma linha não corresponde a nenhuma peça Lego existente]
→ Descrição da nova peça + sugestão de nome + grupo sugerido

FORÇA GERAL: [média das forças] /5
DESTAQUE: [a linha/técnica mais forte do criativo]
FRAQUEZA: [a linha/técnica mais fraca ou ausente]
═══════════════════════════════════════════════
```

---

### ETAPA 4 — SÍNTESE COMPARATIVA

Depois de mapear todos os 3-5 criativos, gerar uma **síntese cruzada**:

```
TEMPLATE DE SÍNTESE:

═══════════════════════════════════════════════
SÍNTESE — BENCHMARK DE CRIATIVOS
Projeto: [nome]
Criativos analisados: [quantidade]
═══════════════════════════════════════════════

PADRÕES RECORRENTES:
→ [O que TODOS os criativos fortes fazem?]
→ Peças que aparecem em 3+ criativos: [lista]
→ Sequência mais comum: [ex: Identificação → Loop → Prova → CTA]

TÉCNICAS DOMINANTES NO NICHO:
→ [Quais técnicas os concorrentes mais usam?]
→ [Quais técnicas NENHUM concorrente usa? = oportunidade]

ÂNGULOS DOS CONCORRENTES:
→ [Quais categorias de ângulo (das 16) estão sendo usadas?]
→ [Quais NÃO estão sendo usadas? = oportunidade de diferenciação]

SUPER ESTRUTURAS USADAS:
→ [Quais autoridades/referências externas os concorrentes usam?]
→ Oportunidade: [super estruturas que SÃO críveis pro público mas ninguém está usando]

NOMES CHICLETE DO MERCADO:
→ [Quais nomes catchy os concorrentes estão usando?]
→ Status: [saturado / em alta / emergente]
→ Oportunidade: [espaço para nome chiclete novo?]

FORMATOS DOMINANTES:
→ [UGC? Médico? Lista? Comparativo?]
→ Formato sub-explorado: [oportunidade]

GAPS IDENTIFICADOS:
→ [O que FALTA nos criativos dos concorrentes?]
→ [Onde nossos criativos podem ser superiores?]

PEÇAS NOVAS PARA CATALOGAR:
→ [Consolidação de todas as peças novas encontradas nos mapas]
→ Para cada: nome sugerido + função + grupo Lego sugerido

RECOMENDAÇÕES PARA CRIAÇÃO:
→ Usar: [técnicas/estruturas que comprovadamente funcionam no nicho]
→ Evitar: [o que está saturado ou não funciona]
→ Inovar: [oportunidades de diferenciação baseadas nos gaps]
═══════════════════════════════════════════════
```

---

### ETAPA 5 — DELETAR E REESCREVER

**Princípio central:** Depois de extrair a estrutura invisível, o texto original é DESCARTADO. O agente NUNCA copia texto de concorrentes. O que se leva é:

```
O QUE DESCARTA:           O QUE LEVA:
─────────────────         ─────────────────
Palavras exatas           Sequência de funções psicológicas
Frases do concorrente     Mapa de peças Lego
Tom específico            Técnicas que funcionam
Dados/provas alheios      Padrões recorrentes do nicho
                          Gaps e oportunidades
                          Peças novas identificadas
```

**Regra inegociável:** O output da Estrutura Invisível alimenta os Passos 1-6 como INTELIGÊNCIA DE MERCADO, nunca como template de cópia. O agente cria usando seus próprios inputs (Agentes 01-10), suas próprias peças Lego e seus próprios dados — mas INFORMADO pela estrutura que funciona no mercado.

---

## REGRAS DE EXECUÇÃO

```
1. NUNCA copiar texto de concorrentes — copiar ESTRUTURA, não PALAVRAS
2. NUNCA usar dados/provas de concorrentes — usar apenas dados dos inputs (Agentes 01-10)
3. Mínimo 3 criativos analisados para que os padrões sejam confiáveis
4. Máximo 5 criativos por rodada — qualidade > quantidade
5. Cada análise deve identificar pelo menos 1 gap/oportunidade
6. Peças novas identificadas devem ser sinalizadas para catalogação futura
7. A síntese comparativa é OBRIGATÓRIA — análises individuais sem síntese = trabalho incompleto
8. Conectores no Disparo de Dopamina = flag automático (se o concorrente usa conector no disparo, anotar como FRAQUEZA)
9. Super Estrutura deve ser avaliada quanto à CREDIBILIDADE para o público-alvo específico
10. Tempo estimado de execução: 1 rodada completa por sessão de benchmark
```

---

## OUTPUT FINAL DO PASSO 0

O output combinado do Passo 0 (Pesquisa Orgânica + Análise de Validados + Estrutura Invisível) é salvo em:

```
outputs/{projeto}/11-ads/benchmark-criativos.md
```

Contém:
1. Dossiê de orgânicos (hooks, linguagem, formatos)
2. Lista de ads validados encontrados
3. Mapas de Estrutura Invisível (3-5 criativos)
4. Síntese comparativa com gaps e recomendações
5. Peças novas identificadas para catalogação

Este arquivo é consultado durante TODOS os passos subsequentes (1-6) como inteligência de mercado.
